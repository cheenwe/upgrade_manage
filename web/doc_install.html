安装与部署

本系统为 Python Flask 应用，支持 Windows / Linux / macOS。通过 .env 配置环境变量，使用 SQLite 存储数据，上传文件存放于指定目录。

环境要求

  • Python：3.8 或以上
  • 依赖见项目根目录 requirements.txt（Flask、Flask-CORS、PyJWT、Werkzeug、python-dotenv）

安装步骤

  1. 将项目克隆或解压到目标目录，进入项目根目录。
  2. （推荐）创建并激活虚拟环境： 
    python -m venv venv
# Windows: venv\Scripts\activate
# Linux/macOS: source venv/bin/activate
  3. 安装依赖： 
    pip install -r requirements.txt
  4. 配置环境变量：在项目根目录创建 .env 文件（可复制 .env.example 后修改），参见下方「配置说明」。
  5. 启动服务： 
    python app.py
    首次运行会自动创建数据库（默认 data/upgrade.db）、上传目录（默认 uploads/）；若数据库中无用户，会使用 .env 中的默认管理员账号创建首个管理员。
  6. 浏览器访问 http://<服务器IP>:<PORT>/（如 http://localhost:8080/），打开登录页即可使用。

配置说明

所有配置通过项目根目录的 .env 文件或系统环境变量设置，键值格式 KEY=value，无需引号。

变量名说明默认值
HOST监听地址，0.0.0.0 表示所有网卡0.0.0.0
PORT监听端口5000
DEBUG是否开启调试（true/false）true
DATABASESQLite 数据库文件路径，留空则使用 data/upgrade.db（项目内 data/upgrade.db）
UPLOAD_ROOT上传文件根目录，留空则使用 uploads/（项目内 uploads/）
SECRET_KEYJWT 及密码加密密钥，生产环境务必修改upgrade-system-secret-key-…
JWT_ALGORITHMJWT 算法HS256
JWT_EXPIRE_DAYS登录 Token 有效天数7
DEFAULT_CLEANUP_MONTHS默认「保留最近 N 个月」历史（系统设置中可改）3
DEFAULT_ADMIN_USERNAME仅当数据库无用户时创建的默认管理员用户名admin
DEFAULT_ADMIN_PASSWORD仅当数据库无用户时创建的默认管理员密码admin123
API_TOKEN用于 /api/upload-by-token 的密钥，脚本/CI 通过 curl 上传升级包；留空则禁用该接口（无）

首次初始化后，用户与密码以后在「用户管理」中维护，修改 DEFAULT_ADMIN_* 仅影响尚未创建任何用户时的首次创建。

curl 上传升级包

在 .env 中配置 API_TOKEN 后，可使用 curl 直接上传（无需登录）：

curl -X POST -H "Authorization: Bearer YOUR_API_TOKEN" \
  -F "file=@./your-package.zip" -F "config_id=1" \
  "https://your-server/api/upload-by-token"

版本管理页每个版本卡片提供「复制上传命令」按钮，可复制带对应 config_id 的 curl 命令，将 YOUR_API_TOKEN 和文件路径替换后即可执行。

运行方式

  • 开发：python app.py，按 Ctrl+C 停止。
  • 生产：建议使用 Gunicorn、uWSGI 等 WSGI 容器，并配合 Nginx 做反向代理与静态资源；同时将 DEBUG=false、修改 SECRET_KEY，并确保 data/、uploads/（或自定义路径）有读写权限。
