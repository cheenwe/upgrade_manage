系统介绍

本系统用于应用/升级包的上传、版本管理与历史记录查询。支持按多种「上传配置」上传文件（如 JAR、ZIP、TAR 等），上传后可执行仅复制、unzip 解压或 tar 解压；版本号按配置与包名自动维护，历史记录支持下载与路径查看。

登录与权限

使用前需在登录页输入用户名与密码。系统分为两种角色：

  • 管理员：可访问全部功能，包括系统设置与用户管理。
  • 普通用户：仅能进行升级相关操作（主页、版本管理、上传新版本、升级历史、文档），无法进入系统设置与用户管理；若直接访问将提示无权限并跳转首页。

被禁用的账号无法登录，会提示「账号已禁用」。

功能模块说明

主页

主页提供快捷入口与各上传配置的当前版本一览。可快速进入版本管理、升级历史、系统设置（管理员可见），以及「快速上传新版本」；下方展示各配置对应的包名与当前版本号。

版本管理

版本管理按「上传配置 + 包名」展示版本表，包括配置名、包名、当前版本号、最后更新时间等，便于查看各业务包的最新版本。

上传新版本

上传新版本：先选择一条上传配置（在系统设置中由管理员维护），再选择本地文件上传。系统会根据配置执行：

  • 仅复制：将文件复制到配置的目标路径；
  • unzip 解压：上传 ZIP 后解压到目标目录；
  • tar 解压：上传 TAR/TAR.GZ 后解压到目标目录。

版本号按该配置与包名自动递增（如 V1.0 → V1.1），无需手动填写。

升级历史

升级历史展示所有上传记录：上传用户、文件名、类型、大小、存储路径、上传时间等。每条记录均支持下载。存储路径默认只显示「倒数第 4 个 /」之后的片段，鼠标悬停可查看完整路径。

系统设置（仅管理员）

系统设置包含：

  • 上传配置：新增/编辑/删除配置，设置配置名、上传路径、上传后操作（仅复制 / unzip / tar）、当前版本号、说明等；
  • 自动清理：设置历史记录保留月数，并可立即执行清理（删除超过保留期的历史记录及对应文件）。

用户管理（仅管理员）

用户管理支持：

  • 新增用户：设置用户名、密码、角色（管理员 / 普通用户）；
  • 编辑用户：修改用户名、角色，可选修改密码；
  • 禁用/启用：禁用后该用户无法登录；
  • 修改密码：单独为某用户重置密码；
  • 删除用户：删除后不可恢复，不能删除当前登录的自己。
